02

Theme posture & where chrome lives

  • Default dark, light toggle in nav. Open dark (marquee), remember the user's choice, honor OS preference. Both themes share geometry, type, and motion — only substrate + foreground invert.
  • Full chrome substrate behind: home hero, partner hero, billing/success. Use all four layers (conic + overlay + top gloss + legibility wash).
  • Quiet substrate (static base + faint corner glow, no rotation) behind: listings grid, listing detail, the application/tours wizards, contact forms. The metal must never sit behind dense fields or tables.

Live reference build: "Axis Hero — Blue Steel (Light + Dark)" — copy its navbar, eyebrow, search bar, and card markup verbatim.

06

Motion & accessibility

  • chromeFlow 24s linear (base conic) and chromeShift 18s ease-in-out (overlay) only on full-chrome heroes. sheen 7s on the search bar. Keep existing animate-page-enter for route transitions.
  • All ambient motion halts under prefers-reduced-motion — freeze to a static frame (the codebase already gates this; extend it to the new keyframes).
  • The legibility wash (layer 4) is required, not decorative — it guarantees ≥4.5:1 for body text over the metal. Keep primary copy in the central calm zone, off the bright top-gloss band.
  • Theme toggle is a real labeled button pair; the filled segment + icon carry state, never color alone.
