01

Admin identity cues

Admins move between this and previewing other portals, so the surface must read unmistakably as "platform". Differentiate without a second accent hue — stay on cobalt, shift the neutrals:

  • Default dark (vs light for managers). Deeper base #0a0e18, cooler/greyer sidebar chrome, slightly lower metal saturation.
  • "Admin" wordmark in the sidebar brand block ("Axis Housing · Admin") plus a small mono ADMIN tag pill near the logo.
  • Preview-as-user banner: when impersonating, a persistent cobalt glass strip at the very top ("Previewing as … · Exit") — reuse PortalTopBanners.

Everything else (nav active states, count badges, content surface, header action) matches the property portal exactly.

03

The review-queue pattern

Properties and leases are decision queues — the admin's core loop. Standardize one pattern:

  • Queue row / card: submission summary on the calm surface, a status badge from the shared palette, and an action cluster.
  • Decision actions: Approve = cobalt primary; Request changes = secondary outline; Reject / Unlist = quiet danger (red text, no fill) — destructive actions never get a loud fill.
  • Filters/tabs: segmented-pill across queue states. Bulk states stay legible on a solid surface.

AxisUsers: one dense table for managers/owners/residents with role chips (cobalt), tier badges, and an active/disabled toggle. "Preview as user" = secondary button that triggers the impersonation banner.

04

Do & don't

DO
  • Mark the surface as admin (wordmark + tag).
  • Keep one review-queue pattern across sections.
  • Use a persistent banner while previewing.
  • Keep destructive actions quiet.
DON'T
  • Don't add an admin-only accent color.
  • Don't make Reject a filled red button.
  • Don't hide that you're impersonating.
  • Don't diverge from the portal shell.
