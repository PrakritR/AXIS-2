01

App shell

Rendered by renderPortalSection("pro", …) with PortalSidebar + PortalTopBanners. The shell is the one place metal appears — confined to the sidebar brand header.

  • Sidebar (266px): brand header carries a contained chrome gradient (linear-gradient(135deg,#2a3c5e,#16233f,#0e1830) dark / #e9eefb→#d7e1f3 light) with a soft radial highlight + the frosted logo tile. Nav below sits on the calm surface.
  • Nav items: 14px, 14px radius. Active = glass fill + inset border + glowing accent dot (dark) / white card + soft shadow + cobalt dot (light). Count badges = metallic pill (dark) / cobalt pill (light).
  • Content surface: dark #0b1120 + a faint cobalt corner glow; light linear-gradient(180deg,#f5f8fd,#e9eef7). Header bar holds the section title + a metallic/cobalt primary action (e.g. Add listing).
  • PortalTopBanners: free-tier upgrade + admin-preview banners become tinted glass strips (cobalt for info, amber for action-needed).

PublicNavbar stays on authenticated areas per current architecture; give it the same glass treatment as marketing but anchored to the active theme.

03

Data components

KPI tile
16px radius, glass fill (rgba(255,255,255,.05) dark / white + soft shadow light). Big number 32px/700, label, small caption. Tiles that need attention take an amber border; neutral tiles take the standard border.
List card
Mono uppercase section label + "View all" accent link, then rows. Each row = name + sub-line + a status badge. Row background is a faint inset fill.
Status badge
Pending = amber, Approved/info = cobalt, Confirmed/paid = green, Overdue/error = red. Pill, 10px/700. Defined once for both themes (see palette below).
Table / ledger
Solid surface only — never over chrome. Mono column headers, hairline rows, faint cobalt hover. Lease pipeline + payment ledger share this. Tabs (pending/approved/rejected) reuse the segmented-pill.
Status palette
Pending Approved Confirmed Overdue Dark variant
04

New-listing wizard

8-step flow with a left step rail, top progress bar, and footer dots. The reference build is the canonical spec — match its rail (active step = numbered cobalt circle + glass pill), progress fill (cobalt→sky), field styling, and stepper control.

Same rail + progress + footer pattern serves the resident-facing flows and the marketing application wizard — build it once as a shared shell.

05

Subscription gating

Free tier hides Residents, Services, Inbox, and account links and shows PortalTierPaywall. Style the paywall as a centered glass card with a chrome accent strip, tier comparison, and a metallic/cobalt upgrade CTA — the one moment chrome may bloom inside the app to signal "premium".

06

Do & don't

DO
  • Confine metal to the sidebar brand header.
  • Keep tables/ledgers on solid surfaces.
  • Reuse the status palette everywhere.
  • Default to light for daily data work.
DON'T
  • Don't animate chrome behind data.
  • Don't invent new status colors.
  • Don't reskin one section in isolation.
  • Don't shrink data text below 13px.
