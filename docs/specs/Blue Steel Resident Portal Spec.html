Resident Portal

The tenant's home base. The same Blue Steel app shell, tuned to feel welcoming and personal — status-led, action-forward, with a nav that grows as the resident's lease progresses.

Read first. Inherits the app shell, status palette, and tables from the Property Portal spec. Live reference: the resident dashboard in "Axis Portal — Blue Steel Dashboards". This sheet covers resident-specific behavior.

01

Tone & identity

  • Default light — warmest, most approachable read; dark available via the same toggle.
  • Personal header: "Welcome, {name}" + property · room · neighborhood sub-line, with the resident's avatar (cobalt→sky gradient initials) top-right. Sidebar brand reads "Resident portal".
  • Status-led dashboard: lead with state banners (approval, lease-to-sign, balance) before the tile grid — the resident's first question is always "what do I need to do?"

Fewer, larger touch targets than the manager portal; this is often used on mobile. Tiles read as friendly cards, not data cells.

02

Dynamic nav by lease state

getResidentPortalDefinition() returns a different nav depending on access state. Style both the same; only the item set changes.

Limited · pre-approval
Dashboard · Payments · Move-in · Inbox · Profile
Full · approved + paid tier
+ Lease · Services (requests + work-orders)

Locked items (not yet available) may appear muted with a small lock glyph rather than disappearing, so residents understand what unlocks next — designer's call per item.

04

Key patterns

Status banners
Approval = green glass, lease-to-sign = cobalt glass, balance due = amber glass. Each pairs a plain-language message with one accent action link. Stack at the top of the dashboard.
Action tiles
Lease / Charges / Maintenance / Inbox / Property / Next payment. Each is a glass card with a mono label, the key value (badge, amount, counts), and a single accent "→" action. Maintenance shows open/scheduled/completed counts color-coded by status.
Pay / sign flows
Primary money/lease actions use the metallic (dark) / cobalt→sky (light) pill. Stripe checkout opens in its own styling inside a glass frame. Autopay setup is a secondary action.
Move-in (gated)
Until the lease is fully signed, show a calm locked state (muted card + lock glyph + "Available once your lease is signed") instead of the room/address details.
05

Manager-tier gating

If the resident's manager is on the Free tier, lease / services / inbox show an upgrade notice instead of full functionality. Frame it gently — a glass card explaining the feature is awaiting the manager, in neutral language (never blame the resident), with no loud CTA.

06

Do & don't

DO
  • Lead with status; surface the next action.
  • Keep targets large (mobile-first).
  • Reuse the shared status palette.
  • Frame gated states gently.
DON'T
  • Don't render dense manager-style tables.
  • Don't blame the resident for tier limits.
  • Don't bury balance/lease actions.
  • Don't diverge from the portal shell.
